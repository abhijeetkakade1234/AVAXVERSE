## 2024-05-24 - [CRITICAL] Push vs Pull / Denial of Service (DoS) in Escrow Dispute Split
**Vulnerability:** Found a DoS vulnerability in `resolveDisputeSplit` within `packages/contracts/contracts/Escrow.sol`. The contract was sending AVAX using `.call{value: ...}('')` to the feeRecipient, client, and freelancer. The contract `require`d the transfer to succeed. If a malicious smart contract acting as client or freelancer implemented a `receive()` function that intentionally reverted, it would completely block the mediator from completing the dispute resolution and thus lock all the funds permanently.
**Learning:** Smart contracts should never `require` the success of external calls to user-controlled addresses when distributing funds to multiple parties, especially when it permanently blocks state progression.
**Prevention:** Always use a withdrawal pattern (Pull over Push) or implement a `_safeTransfer` helper that tracks failed transfers in a `pendingWithdrawals` mapping (as was already correctly done in `_releaseFunds` within the same contract).
