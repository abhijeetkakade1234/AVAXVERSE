## 2024-05-18 - [Fix DoS Vulnerability in Escrow Resolution]
**Vulnerability:** A Denial of Service (DoS) vulnerability was found in the `resolveDisputeSplit` function of `packages/contracts/contracts/Escrow.sol`. The function used strict `require` statements when transferring Ether to the fee recipient, client, and freelancer using `call{value: ...}`. This allowed a malicious client or freelancer contract to intentionally revert when receiving Ether, which would cause the entire `resolveDisputeSplit` transaction to fail, permanently locking the other party's funds in the escrow.
**Learning:** The vulnerability existed because the resolution transaction relied on successful synchronous execution of external calls to all parties. If any single transfer failed, the entire transaction reverted, breaking the resolution flow and enabling griefing attacks.
**Prevention:** To avoid this, always use a pull-over-push pattern for payouts or implement a fallback mechanism like `pendingWithdrawals` that logs failed transfers and allows users to withdraw them manually later. The existing `_safeTransfer` helper function in the contract provides exactly this safeguard and should be used consistently.
